# Contributing to Apollo

Thanks for your interest in improving Apollo! Below is everything you should need in order to get started contributing.

## Code of Conduct

Read the [Code of Conduct](CODE_OF_CONDUCT.md) to get a feel for how I handle my communities.

## Reporting Bugs/Features

If you have a security issue, please follow the directions in [SECURITY](SECURITY.md). Otherwise, you can submit bug report and feature requests [here](https://github.com/codeacula/apollo/issues/new).

## Further Contact

You can either join our community over at [Codeacula's Workshop](https://discord.gg/sTgnMduGCF), or reach out to me directly through my website [Codeacula.com](https://codeacula.com).

## Environment Setup

Follow the directions in the [Local Development](ARCHITECTURE.md#local-development) section of `ARCHITECTURE.md` to set up your local development environment. For AI-assisted development using an isolated container, see the [AI-Assisted Development](ARCHITECTURE.md#ai-assisted-development-container) section.

## Architecture

To gain an understanding of the project's architecture, please refer to [ARCHITECTURE.md](ARCHITECTURE.md).

## Coding Standard & Philosophy

When contributing code, please adhere to the coding standards and philosophies outlined in the [Coding Practices](ARCHITECTURE.md#coding-practices) section of `ARCHITECTURE.md`.

## Pull Requests

When you're ready to submit a Pull Request, please follow the [Commit & Pull Request Guidelines](AGENTS.md#commit--pull-request-guidelines) in `AGENTS.md`.
